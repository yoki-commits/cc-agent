# Code Review Skill

## 使用时机

当用户要求审查代码、检查 Pull Request、评审代码质量时使用此技能。

## 审查流程

1. **整体审视**：理解改动意图，判断是否与项目风格一致
2. **逐文件检查**：
   - 命名规范：变量、函数、类命名是否清晰
   - 逻辑正确性：边界条件、空值检查、异常处理
   - 代码复用：是否重复造轮子，能否抽取公共逻辑
   - 性能问题：是否有不必要的循环、I/O 或内存浪费
3. **安全审查**：检查 SQL 注入、XSS、硬编码密钥等安全问题
4. **输出报告**：按严重程度（阻塞/建议/优化）分类输出

## 输出格式

```
## Code Review 报告

### 🔴 阻塞问题
- [文件:行号] 问题描述 → 建议修改方案

### 🟡 建议改进
- [文件:行号] 问题描述 → 建议修改方案

### 🟢 优化项
- [文件:行号] 优化建议
```
