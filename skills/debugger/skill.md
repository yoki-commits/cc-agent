# Debugger Skill

## 使用时机

当程序报错、出现异常堆栈、运行结果不符合预期时使用此技能。

## 调试流程

1. **阅读错误信息**：提取错误类型、错误消息、堆栈跟踪的关键行
2. **定位问题代码**：根据堆栈找到出错的源文件和行号
3. **分析根因**：检查变量状态、逻辑流程、边界条件
4. **提出修复方案**：给出具体的代码修改建议
5. **验证修复**：确认修改后问题是否解决

## 输出格式

```
## 调试报告

### 错误摘要
- 类型: <错误类型>
- 位置: <文件:行号>
- 消息: <错误消息>

### 根因分析
<分析过程>

### 修复建议
<具体的代码修改>
```
